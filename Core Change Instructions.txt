To make additional core mod changes to forge, this is how it is done.

Setting up Forge within Eclipse:
-------------------------------------------------------------
1. Fork the MinecraftForge repository
2. Clone your repository, and make sure you initialize the FML submodule (git clone --recursive [your URL])
3. git submodule update --init --recursive at {sourceFolder}
4. Run "gradlew setupForge eclipse". This also sets up FML
5. You will have an "eclipse" folder in your Forge root. There will be two projects - "Clean" and "Forge". Make your changes to "Forge"
6. Import existing project in Eclipse, point it towards /projects/minecraftForge, it should display "Clean" and "Forge" import both.
7. This will complete your local install of Forge.

Setting up run configuration for local forge build:
----------------------------------------------------------------------
1. Create new Run Configuration
2. Main CLass: net.minecraft.launchwrapper.Launch
3. Prog Arguments: --version 1.6 --tweakClass cpw.mods.fml.common.launcher.FMLTweaker --accessToken Forge --userProperties {} --username Dev
4. VM Options: -Xincgc -Xmx1024M -Xms1024M -Djava.library.path="c:/projects/almura/build/natives"
5. Working Directory -> Other -> c:/projects/almura/run/client


Build Patch Files and binpatches.pack:
-----------------------------------------------------------
1. Once you've done, gradlew genPatch, run gradlew build
2. It will create some new files at: /projects/minecraftforge/build/distributions/forge-1.7.10-10.13.2.0-universal.jar
3. Within that jar file, pull out binepatches.pack.lzma, put that in the Almura version of modpack.jar
